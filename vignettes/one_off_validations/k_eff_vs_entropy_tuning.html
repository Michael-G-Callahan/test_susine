One-off workbook to calibrate how row entropy maps to an interpretable “effective number of signals” summary for a single alpha row.

We construct synthetic alpha rows with:

  • p = 1000 total variants
  • a high-mass set of size k
  • background_mass spread uniformly across the remaining p - k variants

For example, when k = 2 and background_mass = 0.1, the row is:

  • alpha_1 = alpha_2 = 0.45
  • all other variants get 0.1 / 998

The main tuning question is how the raw row entropy changes as the row behaves more like “mass spread across about k variants.”

2 2. Helpers

xlogx <- function(x) {
  ifelse(x > 0, x * log(x), 0)
}

make_alpha_row <- function(p, k, background_mass) {
  stopifnot(length(p) == 1L, length(k) == 1L, length(background_mass) == 1L)
  stopifnot(k >= 1L, k <= p)
  stopifnot(background_mass >= 0, background_mass < 1)

  if (k == p && background_mass > 0) {
    stop("background_mass > 0 is infeasible when k == p because no background variants remain.")
  }

  alpha <- numeric(p)
  alpha[seq_len(k)] <- (1 - background_mass) / k

  if (k < p && background_mass > 0) {
    alpha[(k + 1):p] <- background_mass / (p - k)
  }

  alpha
}

row_entropy <- function(alpha_row) {
  probs <- alpha_row[alpha_row > 0]
  -sum(probs * log(probs))
}

row_k_eff <- function(alpha_row) {
  exp(row_entropy(alpha_row))
}

compute_row_summary <- function(p, k, background_mass) {
  if (k == p && background_mass > 0) {
    return(NULL)
  }

  main_mass_each <- (1 - background_mass) / k
  background_each <- if (k < p && background_mass > 0) {
    background_mass / (p - k)
  } else {
    0
  }

  entropy <- -(
    k * xlogx(main_mass_each) +
      (p - k) * xlogx(background_each)
  )

  tibble(
    p = p,
    k = k,
    background_mass = background_mass,
    main_mass_each = main_mass_each,
    background_each = background_each,
    entropy = entropy,
    k_eff_entropy = exp(entropy)
  )
}

k_eff_entropy = exp(H) is the entropy-equivalent support size: the number of equally weighted variants that would have the same entropy.

6 6. Why This Is Not One-to-One in k

For fixed background_mass = b, the synthetic row has entropy

\[ H(k, b) = -(1-b)\log\left(\frac{1-b}{k}\right) - b\log\left(\frac{b}{p-k}\right) \]

which can be rearranged as

\[ H(k, b) = (1-b)\log(k) + b\log(p-k) - (1-b)\log(1-b) - b\log(b). \]

This shows the key point:

  • when k increases, the top-set term ((1-b) log k) pushes entropy up
  • but the background term (b log(p-k)) pushes entropy down because the same background mass is being packed into fewer remaining variants

So for b > 0, entropy is generally not monotone in k. It rises, then falls, with its maximum at

\[ k^* = (1-b)p. \]

In this workbook that means:

  • background_mass = 0 peaks at k = 1000
  • background_mass = 0.1 peaks at k = 900
  • background_mass = 0.2 peaks at k = 800

So the current plot should not be expected to be one-to-one in k. Two different k values can give the same entropy once background_mass > 0.

By contrast, if you define

\[ k_{\mathrm{eff}} = \exp(H), \]

then k_eff and entropy are exactly one-to-one, because exp() is a strictly increasing transform.
